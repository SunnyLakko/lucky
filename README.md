# 🔮 Автомат с предсказаниями (Gumball Machine)

Pixel-art веб-автомат с предсказаниями. React + Vite. Работает на мобилках
(touch, drag-and-drop монетки, вибрация) и на десктопе (мышь).
Механика максимально простая: вставил монетку — тут же выпал шарик
с предсказанием, никакой ручки крутить не нужно.

## Запуск локально

```bash
npm install
npm run dev
```

## Деплой на GitHub Pages

1. Создай репозиторий на GitHub, например `gumball-machine`, и запушь код.
2. В `vite.config.js` укажи `base: '/ИМЯ_РЕПОЗИТОРИЯ/'`
   (если это репозиторий вида `username.github.io`, укажи `base: '/'`).
3. Установи `gh-pages` (уже в devDependencies):
   ```bash
   npm install
   ```
4. Задеплой:
   ```bash
   npm run deploy
   ```
   Это выполнит `vite build` (скрипт `predeploy`), а затем опубликует
   содержимое `dist/` в ветку `gh-pages` репозитория.
5. В настройках репозитория GitHub → **Settings → Pages** выбери источник
   **Deploy from a branch → gh-pages → / (root)**.
6. Через пару минут сайт будет доступен по адресу
   `https://<username>.github.io/<ИМЯ_РЕПОЗИТОРИЯ>/`.

## Архитектура данных предсказаний

`src/data/predictions.data.js` — массив из 500 готовых предсказаний
(`{ id, text }`). `src/utils/randomizer.js` берёт случайный элемент
и гарантирует, что он не совпадает с предыдущим показанным (хранит
последний `id` в `sessionStorage` + в памяти на время сессии).

Чтобы добавить ещё предсказаний — просто дополни массив в
`predictions.data.js` объектами вида `{ id: 501, text: "..." }`
(id должен быть уникальным).

## Спрайты (pixel art)

Все спрайты лежат в `public/sprites/` (вырезаны из присланных
изображений, фон сделан прозрачным):

- `machine.png` — корпус автомата (фон `.machine__sprite`)
- `coin.png` — монетка для drag&drop
- `ball.png` — шарик-предсказание (цвет меняется через CSS `hue-rotate`
  в зависимости от id предсказания)
- `slot.png` — прорезь для монетки (декоративная накладка на корпус)
- `tray_bg.png` — лоток, куда падает шарик
- `bg.jpg` — фон всего экрана (сконвертирован в JPEG для лёгкого веса)
- `decor.png` — вывеска "Bubblegum!" над автоматом

Все `<img>` держат `image-rendering: pixelated` (прописано глобально
в `App.css`), поэтому спрайты не размываются при масштабировании.

## Музыкальный плеер

Плеер — компонент `src/components/MusicPlayer.jsx`, плейлист лежит в
`src/data/tracks.data.js`.

**Как добавить свой трек:**
1. Положи mp3-файл в `src/assets/audio/`.
2. Импортируй его в `src/data/tracks.data.js`:
   ```js
   import myTrack from '../assets/audio/my-track.mp3'
   ```
3. Добавь объект в массив `tracks`:
   ```js
   { title: 'Название трека', src: myTrack }
   ```

Как только в массиве окажется больше одного трека, в плеере сами
появятся кнопки "◀ / ▶" для переключения — ничего в коде плеера менять
не нужно.

⚠️ Mp3-файлы существенно увеличивают размер сборки (в проекте уже лежит
~5 МБ трек) — для реального продакшена рекомендуется сжимать аудио
(например, битрейт 128 kbps вместо 320) или подключать треки внешней
ссылкой вместо импорта в бандл.

## Об авторе

Кнопка "?" в правом нижнем углу открывает карточку с автором проекта —
компонент `src/components/AuthorButton.jsx`. Аватар лежит в
`src/assets/author/avatar.jpg`, никнейм и описание захардкожены прямо
в компоненте — правь их там же.

## Обратная связь (Feedback)

Кнопка "✉" в правом верхнем углу открывает форму отзыва —
`src/components/FeedbackButton.jsx`. Отправка сделана через `mailto:`
(без бэкенда): по клику открывается почтовый клиент пользователя с уже
готовой темой и текстом письма.

**Чтобы отзывы реально доходили до тебя**, открой
`src/components/FeedbackButton.jsx` и замени константу:
```js
const FEEDBACK_EMAIL = 'your-email@example.com'
```
на свою настоящую почту. Если хочешь принимать отзывы в Telegram вместо
почты — замени `mailtoHref` на ссылку вида `https://t.me/твой_ник`.
